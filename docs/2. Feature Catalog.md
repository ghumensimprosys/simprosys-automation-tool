# Authentication

## Login

### Purpose

Allow registered users to securely access the platform using email/password or SSO credentials.

### Actors

* Organization Owner
* Admin
* QA Engineer
* Developer
* Viewer

### User Stories

**US-LOGIN-001**
As a registered user, I want to log into the platform so that I can access my projects.

**US-LOGIN-002**
As a user, I want to stay logged in across browser sessions.

### Functional Requirements

#### Login Form

Fields:

| Field       | Type     | Required |
| ----------- | -------- | -------- |
| Email       | Email    | Yes      |
| Password    | Password | Yes      |
| Remember Me | Checkbox | No       |

Actions:

* Login
* Forgot Password

#### Validation

* Email must be valid format.
* Password cannot be empty.
* Display generic error message for invalid credentials.

#### Session Management

* JWT-based authentication.
* Access token expiration: 15 minutes.
* Refresh token expiration: 30 days.
* Automatic token refresh.

### Security Requirements

* Passwords stored using bcrypt.
* Rate limiting on login attempts.
* Account lock after 10 failed attempts.
* CSRF protection.

### Acceptance Criteria

* User can login with valid credentials.
* Invalid credentials show error.
* Session persists when Remember Me is enabled.

## Registration

### Purpose

Allow new organizations and users to create accounts.

### User Stories

**US-REG-001**

As a new customer, I want to create an account so that I can use the testing platform.

### Functional Requirements

#### Registration Form

Fields:

| Field             | Required |
| ----------------- | -------- |
| First Name        | Yes      |
| Last Name         | Yes      |
| Email             | Yes      |
| Password          | Yes      |
| Organization Name | Yes      |

#### Password Rules

* Minimum 8 characters
* One uppercase
* One lowercase
* One number
* One special character

#### Email Verification

* Verification email sent after signup.
* Account inactive until verified.

### Acceptance Criteria

* User receives verification email.
* User can activate account.
* Duplicate emails prevented.

---

## Password Reset

### Purpose

Allow users to recover account access.

### Flow

1. User clicks Forgot Password.
2. User enters email.
3. System sends reset link.
4. User sets new password.
5. Old sessions revoked.

### Requirements

* Reset token expires in 30 minutes.
* Single-use token.
* Audit log entry created.

### Acceptance Criteria

* Password successfully updated.
* Expired links rejected.
