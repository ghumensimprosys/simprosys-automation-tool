# API Specification

## API Standards

### Base URL

```http
https://api.platform.com/v1
```

### Content Type

```http
Content-Type: application/json
```

### Authentication

```http
Authorization: Bearer <jwt_token>
```

### API Versioning

Versioned through URL.

Example:

```http
/v1/projects
/v2/projects
```

---

# Authentication APIs

## Login

### Endpoint

```http
POST /auth/login
```

### Request

```json
{
  "email": "user@example.com",
  "password": "Password123!"
}
```

### Response

```json
{
  "accessToken": "jwt_token",
  "refreshToken": "refresh_token",
  "expiresIn": 900,
  "user": {
    "id": "uuid",
    "email": "user@example.com",
    "role": "Admin"
  }
}
```

### Status Codes

* 200 Success
* 401 Invalid Credentials
* 423 Account Locked

---

## Logout

### Endpoint

```http
POST /auth/logout
```

### Request

```json
{
  "refreshToken": "token"
}
```

### Response

```json
{
  "message": "Logged out successfully"
}
```

# Error Handling

## Standard Error Format

```json
{
  "success": false,
  "error": {
    "code": "PROJECT_NOT_FOUND",
    "message": "Project not found",
    "details": []
  }
}
```

### Common Error Codes

```text
VALIDATION_ERROR
UNAUTHORIZED
FORBIDDEN
NOT_FOUND
CONFLICT
RATE_LIMIT_EXCEEDED
INTERNAL_SERVER_ERROR
```

### HTTP Status Codes

```text
200 OK
201 Created
400 Bad Request
401 Unauthorized
403 Forbidden
404 Not Found
409 Conflict
429 Too Many Requests
500 Internal Server Error
```
